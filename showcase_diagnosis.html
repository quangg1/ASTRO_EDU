🔴 Signal: Section 10 liệt kê chúng là "tham số nên chỉnh đầu tiên" — tức là chúng hay bị lệch.
// Hiện tại: 3 magic numbers không liên quan
ORBIT_DISTANCE_SCALE_AU = ???
ENTITY_SIZE_SCALE_KM = ???
SATELLITE_DISTANCE_COMPRESS = ???

// Đề xuất: 1 BASE_SCALE, các hằng khác là tỉ lệ của nó

② Parent resolution 3 fallback — silent failure

HIGH
Vấn đề: Priority chain parentPlanetName → parentId(planet-*) → catalog. Nếu bước 2 fail silently, entity render sai vị trí không có error.
🟡 Signal: Section 9 item 1 là debug step đầu tiên — tức là parent mapping hay sai.
// Hiện tại: fallback chain im lặng
return parentPlanetName
  ?? resolveFromId(parentId)
  ?? lookupCatalog(entity)

// Đề xuất: resolve rõ ràng + log
function resolveParent(e): ResolvedParent {
  const result = tryAll([
    () => fromName(e.parentPlanetName),
    () => fromId(e.parentId),
    () => fromCatalog(e),
  ])
  if (!result) warn(`[orbit] ${e.id} no parent`)
  return result ?? SUN_FALLBACK
}
✓ Fix: thêm explicit warning + type ResolvedParent để TS enforce.

🔴 Signal: Section 5.4 mô tả bug "chỉ thấy EARTH, scene tối" — camera bị kẹt. Section 11: "reload hard + clear URL param".
// Hiện tại: 3 states + guard patch
idle | transition | focused

  | 'free' // user đang drag tự do

// recovering là state hợp lệ, không phải guard
// free tránh lerp cưỡng bức khi user đang dùng
✓ Fix: thêm 'recovering' và 'free' state. Recovery không conflict với focused.

④ URL camera state — nguồn gốc bug "mất hành tinh"

HIGH
Vấn đề: dist/az/el lưu vào URL. Khi user share link hoặc reload, URL cũ có thể push camera ra vị trí hoàn toàn sai.
🟡 Signal: Section 11 "reload hard + clear URL camera param trước khi kết luận data lỗi".
// Đề xuất: validate URL params trước khi apply
function parseCameraUrl(url): CameraState | null {
  const { dist, az, el } = parseUrl(url)
  // validate bounds
  if (dist < MIN_DIST || dist > MAX_DIST) {
    console.warn('URL dist out of range, ignoring')
    return null // dùng default thay vì crash
  }
  return { dist, az, el }
}
✓ Fix: validate + clamp URL params. Nếu invalid → ignore + dùng default view, không phải debug thủ công.
